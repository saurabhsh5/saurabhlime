### 🤖 AI-Powered Security Workflows (28 tickets)

#### LimeGuard — AI Security Review Platform
**What it is**: Production webhook-driven security service I designed, built, and operate. Orchestrates TruffleHog, Semgrep, and Trivy scanning across all Lime repositories. In 2026, I authored the full architecture RFC to evolve it into an AI-powered PR review platform.

**Production System (Shipped)**:
- Go monorepo (`security/lime-guard/`) — 32 source files, Docker-deployed
- Redis queue-based worker architecture (BRPOP consumers)
- Three security scanners per-push with Slack alerting
- HMAC-SHA256 webhook signature validation

**AI Extension (RFC'd — 7-Phase Architecture)**:
1. **Webhook Extension** ✅ — Route PR events to AI queues
2. **AI Service Foundation** 📋 — TypeScript + AWS Bedrock (Claude Opus), <60s latency target
3. **Rules Service** 📋 — CLAUDE.md parser for context-aware enforcement
4. **Context Service** 📋 — pgvector embeddings (10K files × 1536-dim per repo)
5. **Learning Engine** 📋 — Feedback loop (👍/👎) for team-scoped learning
6. **Visualization** 📋 — Auto-generated Mermaid sequence diagrams
7. **Scale** 📋 — K8s deployment, 99.9% uptime SLO, 100+ PRs/day throughput

**Strategic Positioning**: "Almost equivalent to a security engineer writing a review comment on your PR" — automating secure code review at scale.
